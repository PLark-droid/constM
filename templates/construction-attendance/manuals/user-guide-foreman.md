# 班長・現場監督向け操作ガイド

建設業向け勤怠・日報テンプレート（Lark Bitable） — 班長・現場監督 操作マニュアル

---

## はじめに

本ガイドは、班長（foreman）および現場監督（site supervisor）が日常業務で行う操作手順をまとめたものです。

班長・現場監督の主な役割は以下の通りです。

| 役割 | 主な責任 | 利用端末 |
|------|---------|---------|
| **班長** | 班員の出退勤確認、承認、日報チェック | スマホ + PC |
| **現場監督** | 全体俯瞰、最終承認、集計確認、コスト管理 | PC中心 |

### 班長・現場監督がアクセスできるテーブルと権限

| テーブル | 班長 | 現場監督 |
|---------|------|---------|
| T1: 案件マスタ | 閲覧可 | 閲覧・編集可 |
| T2: 作業員マスタ | 閲覧可（日当単価は非表示） | 閲覧・編集可 |
| T3: 作業内容マスタ | -- | 閲覧・編集可 |
| T4: 出退勤記録 | 閲覧可、承認状態の変更可 | 閲覧・編集可 |
| T5: 日報 | 閲覧可、承認状態の変更可 | 閲覧・編集可 |
| T6: 工数集計 | -- | 閲覧可 |

## 1. 日次の確認作業

毎日行う確認・承認作業の手順を説明します。朝・日中・夕方の3つのタイミングで実施してください。

### 1-1. 出勤状況の確認（毎朝 8:00頃）

朝礼前に、本日の出勤者を確認します。

**操作手順:**

1. Larkアプリを開き、Bitableを選択する
2. 「出退勤記録」テーブル（T4）を開く
3. デフォルトで表示される **「本日の出退勤」ビュー** を確認する
   - このビューは `打刻日 = 本日` かつ `打刻種別 = 出勤` でフィルタ済み

**確認項目:**

| 確認項目 | 見方 | 対応 |
|---------|------|------|
| 出勤者数 | ビューに表示されているレコード数 | 予定人数と一致しているか確認 |
| 遅刻者 | 「出勤ステータス」列が「遅刻」のレコード | 赤文字で表示される。本人に声掛け |
| 直行者 | 「出勤ステータス」列が「直行」のレコード | 現場到着の連絡を待つ |
| 未打刻者 | ビューに名前がない作業員 | 本人に連絡し、打刻忘れか欠勤かを確認 |
| 承認状態 | 「承認状態」列 | 全て「未承認」が正常（出勤直後のため） |

**表示フィールド:**

作業員 / 対象案件 / 打刻種別 / 出勤時刻 / 退勤時刻 / 実働時間(h) / 出勤ステータス / 承認状態

> **ヒント:** 「出退勤カレンダー」ビュー（V4-6）に切り替えると、月間の出退勤状況をカレンダー形式で俯瞰できます。スマホでも直感的に確認できるため、休日出勤や欠勤パターンの把握に便利です。

---

### 1-2. 日報の確認・承認（毎日 17:00頃）

退勤前に、班員が提出した日報を確認・承認します。

#### 日報の承認手順

**操作手順:**

1. 「日報」テーブル（T5）を開く
2. ビューを **「承認待ち」** に切り替える
   - このビューは `承認状態 = 提出済` でフィルタ済み（班長の承認待ちのレコードのみ表示）
3. 一覧に表示されているレコードを1件ずつ確認する

**表示フィールド（承認待ちビュー）:**

報告日 / 報告者 / 対象案件 / 作業内容1 / 作業時間1(h) / 進捗メモ / 安全事項 / 承認状態

**承認する場合:**

1. 承認対象のレコードをタップして詳細画面を開く
2. 以下の内容を確認する
   - **作業内容**: 実際に行った作業と一致しているか
   - **作業時間**: 出退勤時刻と矛盾していないか
   - **安全事項**: ヒヤリハット等の重要報告がないか
   - **写真**: 施工記録として適切か（添付がある場合）
3. 「承認状態」フィールドをタップする
4. 選択肢から **「班長承認」** を選択する
5. 必要に応じて「承認コメント」に一言記入する（任意）
6. 変更は自動保存される

**差戻しする場合:**

1. 「承認状態」フィールドをタップする
2. 選択肢から **「差戻し」** を選択する
3. **「承認コメント」フィールドに差戻し理由を必ず記入する**
   - 例: 「作業時間の合計が実働時間と合っていません。確認して再提出してください」
   - 例: 「安全事項の記入をお願いします」
4. 変更は自動保存される
5. 報告者に差戻し通知が自動送信される

> **重要:** 差戻しの場合は必ず承認コメントに理由を記入してください。理由がないと報告者がどこを修正すべきか分かりません。

#### 未提出者の確認

1. ビューを **「未提出確認」** に切り替える
   - このビューは `承認状態 = 未提出 または 差戻し` でフィルタ済み
2. 未提出のレコードがあれば、該当する作業員に提出を促す
   - 古い日付から表示されるため、長期間未提出のものが上部に来る

### 1-3. 出退勤の承認（毎日 17:00〜翌朝）

班員の出退勤記録を承認します。退勤打刻が完了してから承認を行ってください。

**操作手順:**

1. 「出退勤記録」テーブル（T4）を開く
2. ビューを **「未承認一覧」** に切り替える
   - このビューは `承認状態 = 未承認` かつ `打刻種別 = 出勤` でフィルタ済み
   - 古い日付順に表示されるため、承認漏れを防止できる

**表示フィールド（未承認一覧ビュー）:**

打刻日 / 作業員 / 対象案件 / 出勤時刻 / 退勤時刻 / 実働時間(h) / 人工数 / 出勤ステータス / 承認状態 / 備考

**承認する場合:**

1. 各レコードの出勤時刻・退勤時刻・実働時間を目視確認する
2. 「承認状態」フィールドをタップする
3. 選択肢から **「班長承認」** を選択する
4. 変更は自動保存される

**時刻を修正してから承認する場合:**

1. レコードをタップして詳細画面を開く
2. 出勤時刻または退勤時刻を正しい値に修正する
3. 実働時間(h)と人工数は自動で再計算される
4. 「承認状態」を「班長承認」に変更する

**差戻しする場合:**

1. 「承認状態」を **「差戻し」** に変更する
2. 「備考」フィールドに差戻し理由を記入する
   - 例: 「退勤時刻が入力されていません。退勤打刻を行ってください」

### 承認フローの全体像

```
出退勤記録（T4）:
  [未承認] ──(班長が承認)──> [班長承認] ──(監督が承認)──> [監督承認]
     ^                          |                          |
     └──── [差戻し] <───────────┘                          |
     ^                                                     |
     └──── [差戻し] <─────────────────────────────────────┘

日報（T5）:
  [未提出] -> [提出済] ──(班長が承認)──> [班長承認] ──(監督が承認)──> [監督承認]
                 ^                          |                          |
                 └──── [差戻し] <───────────┘                          |
                 ^                                                     |
                 └──── [差戻し] <─────────────────────────────────────┘
```

## 2. ダッシュボードの活用

### 2-1. 現場日次ダッシュボード（D1）

現場監督・班長が今日の現場状況をリアルタイムで把握するためのダッシュボードです。

**アクセス方法:**

1. Larkアプリを開き、Bitableを選択する
2. 左側メニュー（またはタブ）から **「現場日次ダッシュボード」** を選択する
3. 期間フィルタが「本日」になっていることを確認する

**確認すべき指標（ウィジェット）:**

| ウィジェット | 内容 | 確認ポイント |
|------------|------|------------|
| **本日の出勤者数**（数値カード） | 今日出勤した作業員のユニーク数 | 予定人数と一致しているか。前日比の増減矢印も確認 |
| **出勤状況一覧**（テーブル） | 作業員名・案件・出退勤時刻・ステータスの一覧 | 未承認レコード（黄色背景）、差戻し（赤背景）、遅刻（赤文字）を確認 |
| **案件別出勤者数**（棒グラフ） | 案件ごとの出勤人数を棒グラフで表示 | 人員配置が計画通りか確認。極端に少ない案件がないか |
| **日報提出状況**（円グラフ） | 提出済み / 未提出の割合をドーナツ型で表示 | 中央の提出率が100%に近いか確認 |
| **未承認件数**（数値カード） | 出退勤と日報の未承認レコード数 | 0件が理想。件数がある場合は速やかに承認処理を実施 |
| **天候**（テキスト） | 本日の天候 | 悪天候時の安全対策を検討 |
| **安全事項ハイライト**（テーブル） | 日報で安全事項が記入されたレコードの一覧 | 「事故」「怪我」「危険」を含む場合は赤背景・太字で表示される。要注意 |

**異常値の見方:**

- 出勤者数が予定より少ない → 欠勤者・遅刻者を確認し、対応を検討
- 未承認件数が増えている → 承認処理が滞っている。速やかに処理を実施
- 安全事項にハイライト表示がある → 即座に内容を確認し、必要に応じて対策を実施

**推奨確認タイミング:**

| 時刻 | 確認内容 |
|------|---------|
| 毎朝 8:00 | 出勤者数・出勤状況一覧で朝礼準備 |
| 毎日 17:00 | 日報提出状況・未承認件数で当日の締め作業 |

### 2-2. 月次管理ダッシュボード（D2）（現場監督向け）

月間の勤怠・工数・コストを把握するためのダッシュボードです。主に現場監督・管理者が利用します。

> **注意:** このダッシュボードは班長には閲覧権限がありません。現場監督以上の役職が対象です。

**アクセス方法:**

1. Bitableの左側メニューから **「月次管理ダッシュボード」** を選択する
2. 期間フィルタで対象年月を選択する（デフォルトは今月）

**確認すべき指標:**

| ウィジェット | 内容 | 確認ポイント |
|------------|------|------------|
| **月間総人工数**（数値カード） | 当月の全案件の人工数合計 | 前月比で大きな増減がないか |
| **月間総労務費**（数値カード） | 当月の労務費合計（円） | 予算内に収まっているか |
| **日別出勤推移**（折れ線グラフ） | 日ごとの出勤人数推移 | 極端な落ち込み・増加がないか。平均線との乖離を確認 |
| **案件別人工数**（積み上げ棒グラフ） | 案件ごとの人工投入量を作業員別に色分け | 特定案件に人員が集中していないか |
| **作業員別出勤日数**（横棒グラフ） | 作業員ごとの出勤日数ランキング | 出勤日数に偏りがないか確認 |
| **職種別人工分布**（円グラフ） | 職種ごとの人工数割合 | 必要な職種が不足していないか |
| **案件別労務費**（テーブル） | 案件ごとの人工数・金額一覧 | 金額が最大の案件は赤背景で表示。予算消化率を確認 |
| **長時間勤務アラート**（テーブル） | 実働10時間超のレコード一覧 | 12時間超は赤背景・太字。該当者への対応が必要 |

**人工数推移の見方:**

- 折れ線グラフで日ごとの出勤人数を確認
- 点線の補助線は月間平均出勤人数
- 平均を大きく下回る日がある場合、人員不足やスケジュールの問題を調査

**案件別コスト確認:**

- 「案件別労務費」テーブルで、案件ごとの総人工数と金額を確認
- 総人工数が50以上の案件は太字表示
- 金額が最大の案件は背景色で強調表示

**長時間勤務のチェック:**

- 実働時間が10時間を超えるレコードが自動抽出される
- 10h超〜12h: 薄い橙色の背景（注意レベル）
- 12h超: 赤背景・太字（警告レベル、即対応が必要）

### 2-3. 案件別詳細ダッシュボード（D3）（現場監督向け）

特定の案件に絞って、人工投入状況を詳細に確認するためのダッシュボードです。

**アクセス方法:**

1. Bitableの左側メニューから **「案件別詳細ダッシュボード」** を選択する
2. 「案件選択フィルタ」で確認したい案件をドロップダウンから選択する
   - 進行中の案件のみ選択肢に表示される

**確認すべき指標:**

| ウィジェット | 確認ポイント |
|------------|------------|
| **案件情報** | 案件名・発注者・工期・ステータス・現場監督を確認 |
| **累計人工数** | 案件開始からの累計人工数。予算人工と比較 |
| **累計労務費** | 案件開始からの累計労務費。予算との差異を確認 |
| **日別人工推移** | 日ごとの人工投入量の推移。平均補助線との乖離を確認 |
| **作業内容別工数** | どの作業にどれだけ工数を投入しているか |
| **作業員別投入人工** | 誰がどれだけこの案件に入っているか |

## 3. 通知への対応

システムから届く通知と、その対応方法を説明します。

### 3-1. 承認依頼通知

**通知内容:** 職人がフォームから出退勤打刻や日報を提出すると、班長に承認依頼の通知が届きます。

**対応手順:**

1. Larkの通知を確認する
2. 通知をタップして該当テーブルを開く
3. 「承認待ち」ビュー（日報の場合）または「未承認一覧」ビュー（出退勤の場合）で内容を確認する
4. 本ガイドの「1-2. 日報の確認・承認」または「1-3. 出退勤の承認」の手順に従って処理する

**推奨:** 通知が届くたびに即座に対応するのが理想ですが、少なくとも1日1回（夕方17:00頃）にまとめて処理してください。

---

### 3-2. 長時間勤務アラート

**通知内容:** 実働時間が12時間を超えるレコードが登録されると、アラート通知が届きます。

**対応手順:**

1. 通知内容を確認し、該当する作業員と案件を把握する
2. 月次管理ダッシュボード（D2）の「長時間勤務アラート」ウィジェットで詳細を確認する
3. 本人に状況を確認する
   - 実際に長時間勤務をしたのか
   - 打刻ミス（退勤打刻忘れで翌日打刻した等）ではないか
4. 実際の長時間勤務の場合
   - 翌日以降のシフトを調整し、是正措置を講じる
   - 必要に応じて休日を設定する
5. 打刻ミスの場合
   - 本ガイドの「4-1. 出退勤データの修正」に従って時刻を修正する

---

### 3-3. GPS異常検知

**通知内容:** 出退勤打刻時のGPS位置が、案件マスタに設定された現場GPS座標から許容半径（デフォルト500m）以上離れている場合に通知されます。

**対応手順:**

1. 通知内容を確認し、該当する作業員・案件・打刻時刻を把握する
2. 出退勤記録テーブル（T4）で該当レコードを開き、GPS座標を確認する
3. 本人に確認する
   - 直行・直帰で別の場所から打刻した場合 → 出勤ステータスを「直行」「直帰」に変更
   - GPSの精度不良（屋内・地下等）の場合 → 備考にその旨を記入し、承認
   - 不正打刻の疑いがある場合 → 現場監督・管理者にエスカレーション

## 4. データの修正

### 4-1. 出退勤データの修正

班長は出退勤記録の修正権限を持っています。以下のケースで修正を行います。

#### 打刻忘れへの対応（レコード手動追加）

職人が出勤打刻を忘れた場合、班長がレコードを手動で追加できます。

**操作手順:**

1. 「出退勤記録」テーブル（T4）を開く
2. 「本日の出退勤」ビューを表示する
3. テーブル末尾の **「+」ボタン（行追加）** をタップする
4. 新しいレコードに以下を入力する

| フィールド | 入力内容 |
|----------|---------|
| 作業員 | 該当する作業員をリストから選択 |
| 対象案件 | 該当する案件をリストから選択 |
| 打刻種別 | 「出勤」を選択 |
| 打刻日 | 該当日の日付を選択 |
| 出勤時刻 | 実際の出勤時刻を入力 |
| 出勤ステータス | 通常（または該当するステータス） |
| 承認状態 | 「班長承認」を選択（班長が手動追加するため即承認） |
| 備考 | 「班長代理入力（打刻忘れ）」と記入 |

5. 退勤打刻も忘れている場合は、同じレコードの退勤時刻にも値を入力する

> **重要:** 手動追加した場合は必ず備考に理由を記入してください。監査時の証跡となります。

#### 時刻の修正

出勤時刻や退勤時刻に誤りがある場合の修正方法です。

**操作手順:**

1. 修正対象のレコードをタップして詳細画面を開く
2. 出勤時刻または退勤時刻のフィールドをタップする
3. 時刻ピッカーで正しい時刻を選択する
4. 実働時間(h)と人工数は自動で再計算される
5. 備考に修正理由を記入する（例: 「退勤時刻修正: 17:00 -> 17:30（打刻ミス）」）

#### 代理打刻の方法

職人のスマホが故障した場合等、班長が代わりに打刻を行う方法です。

**操作手順:**

1. Larkアプリでブックマーク「出勤打刻」フォーム（F1）を開く
2. 「作業員」フィールドで、代理対象の作業員を選択する（自分ではなく対象者を選ぶ）
3. 「対象案件」で本日の現場を選択する
4. 出勤時刻を確認（現在時刻がデフォルト）
5. 「備考」に **「班長代理打刻」** と記入する
6. 送信する
7. 退勤時も同様に、退勤打刻フォーム（F2）で代理打刻を行う

> **注意:** 代理打刻は必ず備考にその旨を記入してください。GPS位置情報は班長の端末の位置が記録されます。

### 4-2. 日報の修正依頼（差戻しフロー）

日報の内容に不備がある場合は、差戻しフローを使って報告者に修正を依頼します。

**差戻しの操作手順:**

1. 「日報」テーブル（T5）の「承認待ち」ビューを開く
2. 修正が必要なレコードをタップして詳細画面を開く
3. 不備の内容を確認する（よくある不備は以下の通り）
   - 作業時間の合計が実働時間と大きく異なる
   - 作業内容の選択が実際と異なる
   - 安全事項が記入されていない（KY活動日に未記入等）
   - 写真の添付が求められているが未添付
4. 「承認状態」フィールドをタップし、**「差戻し」** を選択する
5. 「承認コメント」フィールドに具体的な修正指示を記入する
   - 悪い例: 「修正してください」（何を修正すべきか不明）
   - 良い例: 「作業時間1が3.0hですが、出退勤記録では実働8.0hです。残りの時間の作業内容2・3も記入してください」
6. 変更は自動保存される
7. 報告者に差戻し通知が自動送信される

**報告者が再提出した後の対応:**

1. 再提出されると承認状態が「提出済」に戻り、再度「承認待ち」ビューに表示される
2. 修正内容を確認し、問題なければ「班長承認」に変更する

## 5. 週次・月次の業務

### 5-1. 週次レビュー（毎週月曜日推奨）

前週の稼働実績を振り返ります。

**操作手順:**

1. 「出退勤記録」テーブル（T4）を開く
2. ビューを **「週次出退勤」** に切り替える
   - 今週分が表示される（前週分を確認する場合はフィルタの日付範囲を手動で変更）

**表示フィールド:**

打刻日 / 作業員 / 対象案件 / 出勤時刻 / 退勤時刻 / 実働時間(h) / 人工数 / 出勤ステータス / 承認状態

**確認項目:**

| 確認項目 | 見方 | アクション |
|---------|------|----------|
| 未承認レコードの有無 | 承認状態が「未承認」のもの | 速やかに承認処理を実施 |
| 遅刻・早退の傾向 | 出勤ステータスが「遅刻」「早退」のもの | 常習的であれば本人に注意 |
| 案件別の人工投入状況 | 対象案件列を確認 | 計画と乖離していれば翌週のシフトを調整 |
| 長時間勤務 | 実働時間が10h超のレコード | 該当者のシフトを調整 |

**現場監督向け追加確認:**

- 案件別詳細ダッシュボード（D3）で、担当案件の日別人工推移を確認
- 月次管理ダッシュボード（D2）の日別出勤推移グラフで全体傾向を確認

---

### 5-2. 月次集計の確認（月末）

月末に当月の工数・コストを確認します。

**工数集計テーブルの確認（現場監督向け）:**

1. 「工数集計」テーブル（T6）を開く
2. 「月次集計一覧」ビューで対象年月のデータを確認する
   - 対象年月・対象案件の2段グループ化で表示される
   - グループのサマリ行で月別・案件別の合計が自動表示される

**表示フィールド:**

対象年月 / 作業員 / 対象案件 / 出勤日数 / 総実働時間(h) / 総人工数 / 金額(円)

3. 「案件別集計」ビューに切り替えると、案件ごとの合計を確認できる

**ダッシュボードでのコスト把握（現場監督向け）:**

1. 月次管理ダッシュボード（D2）を開く
2. 期間フィルタで対象年月を選択する
3. 以下を確認する
   - 月間総人工数 / 月間総労務費（数値カード）
   - 案件別人工数の分布（積み上げ棒グラフ）
   - 案件別労務費（テーブル）
   - 前月比の増減

> **注意:** 工数集計テーブル（T6）は月末のバッチ処理または手動で集計レコードが作成されます。T6にデータがない場合、ダッシュボードの労務費系ウィジェットは0と表示される場合があります。管理者に集計処理の実行を依頼してください。

## 6. よくある質問

### 出退勤に関する質問

| 質問 | 回答 |
|------|------|
| 職人が打刻を忘れた | 班長がレコードを手動追加できます。「4-1. 打刻忘れへの対応」の手順に従ってください。備考に「班長代理入力（打刻忘れ）」と記入してください |
| 退勤打刻をしていないレコードがある | 本人に退勤打刻を促してください。退勤打刻がないと実働時間が計算されません。やむを得ない場合は班長が退勤時刻を手動入力できます |
| 出勤時刻を間違えて打刻した | 班長がレコードを開いて時刻を修正できます。修正理由を備考に記入してください |
| 職人のスマホが使えない | 班長が代理でフォームから入力できます。「4-1. 代理打刻の方法」を参照してください。備考に「班長代理打刻」と記入してください |
| 同じ人が2回出勤打刻してしまった | 重複レコードが発生した場合、管理者に連絡して不要なレコードを削除してもらってください |
| 直行・直帰の場合はどうする？ | 通常通り打刻し、出勤ステータスを「直行」「直帰」「直行直帰」に変更してください。GPS位置が現場から離れていても問題ありません |

### 日報に関する質問

| 質問 | 回答 |
|------|------|
| 日報の内容が不十分 | 差戻しして再提出を依頼してください。「4-2. 日報の修正依頼」の手順に従い、承認コメントに具体的な修正指示を記入してください |
| 職人が前日の日報を出し忘れた | 日報提出フォーム（F3）で「報告日」を前日に変更して提出してもらってください。デフォルトは本日ですが、変更可能です |
| 承認を取り消したい | 承認状態は変更可能です。「班長承認」を「未承認」（出退勤）や「提出済」（日報）に戻すことができます |
| 安全事項に重要な報告がある | ダッシュボード（D1）の「安全事項ハイライト」で確認できます。「事故」「怪我」「危険」を含む報告は赤背景で強調表示されます。必要に応じて即座に対応してください |

### ダッシュボードに関する質問

| 質問 | 回答 |
|------|------|
| ダッシュボードの数値が0と表示される | フィルタの日付範囲を確認してください。対象期間にデータが存在しない場合は0と表示されます |
| 労務費が計算されない | 工数集計テーブル（T6）の集計処理が未実行の可能性があります。管理者に連絡してください |
| グラフが表示されない | 対象期間にデータが0件の場合、グラフは表示されません。フィルタの期間設定を確認してください |

### 権限に関する質問

| 質問 | 回答 |
|------|------|
| 日当単価を確認したい | 班長には日当単価は非表示に設定されています。確認が必要な場合は管理者に問い合わせてください |
| 月次管理ダッシュボードが見られない | 月次管理ダッシュボード（D2）と案件別詳細ダッシュボード（D3）は現場監督以上の権限が必要です |
| 他の班のデータを確認したい | 出退勤記録・日報テーブルでは全データが閲覧可能です。ビューのフィルタ条件を手動で変更してください |
