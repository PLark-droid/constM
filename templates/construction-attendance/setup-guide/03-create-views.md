# 03. ビュー構築手順

> **所要時間**: 約60〜90分（全20ビュー）
> **前提条件**: [02-create-tables.md](./02-create-tables.md) の全テーブル・フィールド作成が完了していること

## 概要

各テーブルにロール別・用途別のビューを作成する手順です。
本テンプレートでは合計 **20ビュー** を作成します。ビューによって表示フィールド・フィルタ・ソート・グループ化を制御し、職人・班長・管理者それぞれに最適な画面を提供します。

### ビュー数一覧

| テーブル | ビュー数 | ビュー名 |
|---------|---------|---------|
| T1: 案件マスタ | 3 | 全案件一覧 / 進行中案件 / 案件カレンダー |
| T2: 作業員マスタ | 3 | 全作業員一覧 / 稼働中作業員 / 職種別一覧 |
| T3: 作業内容マスタ | 1 | 作業内容一覧 |
| T4: 出退勤記録 | 6 | 本日の出退勤 / 週次出退勤 / 未承認一覧 / 個人別月次 / 案件別日次 / 出退勤カレンダー |
| T5: 日報 | 5 | 本日の日報 / 未提出確認 / 承認待ち / 案件別日報 / 日報ギャラリー |
| T6: 工数集計 | 2 | 月次集計一覧 / 案件別集計 |
| **合計** | **20** | |

## ビュー作成の基本操作

すべてのビュー作成に共通する基本手順を先に説明します。

### 新規ビューの追加方法

1. 対象テーブルを開く
2. テーブル上部のビュータブ右にある「**+**」をクリック
3. ビュー種類を選択する（グリッド / カンバン / カレンダー / ギャラリー）
4. ビュー名を入力して確定する

### フィルタの設定方法

1. ツールバーの「**フィルタ**」アイコンをクリック
2. 「**+ 条件を追加**」をクリック
3. フィールド名、条件（=, !=, 含む 等）、値を指定する
4. 複数条件の場合は「AND」「OR」の組み合わせを選択する

### ソートの設定方法

1. ツールバーの「**ソート**」アイコンをクリック
2. 「**+ ソート条件を追加**」をクリック
3. ソートするフィールドと方向（昇順/降順）を選択する
4. 複数ソート条件は上から優先的に適用される

### グループ化の設定方法

1. ツールバーの「**グループ化**」アイコンをクリック
2. 「**+ グループ化を追加**」をクリック
3. グループ化するフィールドを選択する
4. 第2グループを追加する場合は再度「+ グループ化を追加」をクリック

### フィールドの表示/非表示切替

1. ツールバーの「**フィールド設定**」（目のアイコン）をクリック
2. フィールド一覧が表示される
3. トグルスイッチで表示/非表示を切り替える
4. フィールドをドラッグして表示順を変更する

## T1: 案件マスタのビュー（3ビュー）

### V1-1: 全案件一覧（グリッド）-- デフォルト

> **ビュー種類**: グリッド | **デフォルト**: Yes | **主な利用者**: 管理者

テーブル作成時に自動で作成される最初のビューを「全案件一覧」として設定します。

**手順:**

1. 案件マスタテーブルを開く
2. デフォルトで表示されているビュー名（「グリッドビュー」等）をダブルクリックし、「**全案件一覧**」に変更する
3. フィールド設定を開き、以下の表示順・表示/非表示を設定する

**表示フィールド（左から順に）:**

| 順番 | フィールド名 |
|------|-------------|
| 1 | 案件ID |
| 2 | 案件名 |
| 3 | ステータス |
| 4 | 現場住所 |
| 5 | 発注者 |
| 6 | 工期開始 |
| 7 | 工期終了 |
| 8 | 現場監督 |
| 9 | 備考 |

**非表示にするフィールド:**
- 現場GPS緯度
- 現場GPS経度
- GPS許容半径(m)

4. ソートを設定する
   - 第1条件: 「ステータス」昇順（準備中 > 進行中 > 完了 > 中止）
   - 第2条件: 「案件名」昇順

5. フィルタ: 設定なし（全件表示）

---

### V1-2: 進行中案件（カンバン）

> **ビュー種類**: カンバン | **デフォルト**: No | **主な利用者**: 管理者、班長

**手順:**

1. テーブル上部の「**+**」をクリックし、「**カンバンビュー**」を選択する
2. ビュー名に「**進行中案件**」と入力する
3. カンバンの**グループ化フィールド**に「**ステータス**」を選択する
   - 列として「準備中」「進行中」「完了」が表示される
4. フィルタを設定する
   - フィールド: 「ステータス」
   - 条件: 「次と等しくない」
   - 値: 「中止」
5. ソートを設定する
   - 「工期開始」昇順
6. カードに表示するフィールドを設定する
   - カード設定を開き、以下のフィールドを表示にする

**カード表示フィールド:**
- 案件名
- 現場住所
- 工期開始
- 工期終了
- 現場監督

> **活用ポイント**: カード をドラッグ＆ドロップすることでステータスを変更できます。スマホでもスワイプで確認しやすいビューです。

### V1-3: 案件カレンダー（カレンダー）

> **ビュー種類**: カレンダー | **デフォルト**: No | **主な利用者**: 管理者、班長

**手順:**

1. テーブル上部の「**+**」をクリックし、「**カレンダービュー**」を選択する
2. ビュー名に「**案件カレンダー**」と入力する
3. カレンダー設定で日付フィールドを指定する
   - **開始日**: 「工期開始」（`fld_start_date`）を選択
   - **終了日**: 「工期終了」（`fld_end_date`）を選択
4. フィルタを設定する
   - フィールド: 「ステータス」
   - 条件: 「次と等しくない」
   - 値: 「中止」
5. カレンダー上のバーに表示されるフィールドを確認する
   - 案件名
   - ステータス

> **活用ポイント**: 工期の重複・空き期間を月単位で視覚的に確認できます。リソース配分の計画に活用してください。

## T4: 出退勤記録のビュー（6ビュー）

### V4-1: 本日の出退勤（グリッド）-- デフォルト

> **ビュー種類**: グリッド | **デフォルト**: Yes | **主な利用者**: 班長、管理者

今日の出退勤状況をリアルタイムで確認するためのビューです。テーブルを開いたとき最初に表示されます。

**手順:**

1. 出退勤記録テーブルを開く
2. デフォルトビューの名前を「**本日の出退勤**」に変更する
3. フィルタを設定する（2条件、AND）
   - **条件1**:
     - フィールド: 「打刻日」
     - 条件: 「次と等しい」
     - 値: 「**今日**」（動的値 `TODAY()` を選択）
   - 「**AND**」を選択（両方の条件を満たすレコードのみ表示）
   - **条件2**:
     - フィールド: 「打刻種別」
     - 条件: 「次と等しい」
     - 値: 「出勤」

> **フィルタ設定の補足**: Lark Bitableのフィルタで日付フィールドを選択すると、値の入力欄に「今日」「昨日」「今週」「今月」等の**動的値**が選択できます。固定日付ではなく「今日」を選ぶことで、毎日自動的にフィルタが更新されます。

4. フィールド設定を開き、以下の通り設定する

**表示フィールド（左から順に）:**

| 順番 | フィールド名 |
|------|-------------|
| 1 | 作業員 |
| 2 | 対象案件 |
| 3 | 打刻種別 |
| 4 | 出勤時刻 |
| 5 | 退勤時刻 |
| 6 | 実働時間(h) |
| 7 | 出勤ステータス |
| 8 | 承認状態 |

**非表示にするフィールド:**
- 打刻ID
- 打刻日（フィルタで今日に限定しているため不要）
- 出勤GPS
- 退勤GPS
- 休憩時間(h)
- 人工数
- 天候
- 備考

5. ソートを設定する
   - 「出勤時刻」昇順

## Tips

### 共有ビューと個人ビューの使い分け

- **共有ビュー**: 全メンバーに表示されるビュー。本ガイドで作成するビューはすべて共有ビューです
- **個人ビュー**: 自分だけに表示されるビュー。各自で自由にフィルタ・ソートをカスタマイズ可能
- ビュー名の右にある「...」メニューから「個人ビューに変換」「共有ビューに変換」を切り替えられます

### ビューの共有範囲設定

- ビュー右上の「...」メニューから共有範囲を設定できます
- 「誰が見られるか」を制限することで、管理者専用のビューを作れます
- 日当単価が表示されるビュー（全作業員一覧、月次集計一覧等）は管理者のみに共有することを推奨します

### スマホでのビュー切替方法

1. Larkモバイルアプリでビータブルを開く
2. 画面上部のビュー名をタップする
3. ビュー一覧が表示されるので、切替先のビューをタップする
4. カレンダービュー・カンバンビューはスマホでも視認性が高く、おすすめです

### ビューのコピー機能

- 似た設定のビューは「ビューをコピー」機能で効率的に作成できます
- 例: 「本日の出退勤」をコピーして「週次出退勤」のベースにする
- ビュー名の右にある「...」メニューから「ビューをコピー」を選択し、フィルタ等を調整する方法が手早いです

### フィールド幅の調整

スマホでの閲覧を考慮し、以下のフィールド幅を目安に設定してください。

| フィールドの種類 | 推奨幅 |
|-----------------|--------|
| テキスト（氏名等） | 120px |
| 日付 | 100px |
| 数値 | 80px |
| 単一選択 | 100px |
| リンク | 120px |
