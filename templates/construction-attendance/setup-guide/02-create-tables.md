# 02. テーブル構築手順

建設業向け勤怠・日報テンプレート — 全6テーブル・全フィールド作成ガイド

---

## 概要

本ガイドでは、Lark Bitable 上に6つのテーブルを作成し、すべてのフィールドを定義します。テーブル間にはリンク（リレーション）があるため、依存関係のない順序で作成していきます。

**所要時間:** 約60〜90分（初回）

**前提:** [01-create-base.md](./01-create-base.md) でベースの作成が完了していること。

---

## 作成順序

テーブルは以下の順番で作成してください。リンクフィールドの参照先が先に存在している必要があります。

| 順番 | テーブル名 | 理由 |
|------|-----------|------|
| 1 | T1: 案件マスタ | 他テーブルからリンクされる基盤テーブル。リンク先なし |
| 2 | T3: 作業内容マスタ | 他テーブルからリンクされる基盤テーブル。リンク先なし |
| 3 | T2: 作業員マスタ | T1（案件マスタ）へのリンクあり |
| 4 | T4: 出退勤記録 | T1, T2 へのリンクあり |
| 5 | T5: 日報 | T1, T2, T3 へのリンクあり |
| 6 | T6: 工数集計 | T1, T2 へのリンクあり。T2の日当単価をルックアップ |

> **注意:** T3（作業内容マスタ）はリンク先を持たないため、T1の直後に作成します。T2よりも先に作成することで、後のテーブル作成がスムーズになります。

## 共通操作：フィールドの追加方法

すべてのテーブルで共通する操作です。初めに確認しておいてください。

### フィールドを追加する

1. テーブルのグリッドビュー右端にある **「+」ボタン** をクリック
2. フィールドの型（テキスト、数値、日付 等）を選択
3. フィールド名を入力
4. 必要に応じて詳細設定を行う
5. **「確定」** をクリック

### フィールドの順序を変更する

- フィールド名（列ヘッダー）をドラッグ＆ドロップして並べ替え可能

### フィールド名・設定を変更する

- 列ヘッダーのフィールド名をダブルクリック、または右クリック →「フィールドを編集」

### 数式フィールド「実働時間(h)」の設定（詳細）

実働時間を自動計算する数式です。出勤時刻と退勤時刻の差から休憩時間を引きます。

**手順:**
1. 「+」ボタン → 型一覧から **「数式」** を選択
2. フィールド名に `実働時間(h)` と入力
3. 数式入力エリアに以下の数式をコピー＆ペースト

```
IF(
  AND(出勤時刻, 退勤時刻),
  MAX(
    (HOUR(退勤時刻) * 60 + MINUTE(退勤時刻)
     - HOUR(出勤時刻) * 60 - MINUTE(出勤時刻)) / 60
    - 休憩時間(h),
    0
  ),
  0
)
```

4. 「数値の形式」を **「小数」** に設定、小数点以下の桁数を **1** に設定
5. 「確定」をクリック

> **数式の解説:**
> - `AND(出勤時刻, 退勤時刻)` : 出勤時刻と退勤時刻の両方が入力されているかチェック
> - `HOUR()` / `MINUTE()` : 時刻から時・分を取得
> - 退勤の分換算 - 出勤の分換算 → 分単位の差分を60で割って時間に変換
> - `- 休憩時間(h)` : 休憩時間を差し引く
> - `MAX(..., 0)` : マイナスにならないよう0を下限に設定
> - 出勤・退勤いずれか未入力の場合は 0 を返す

> **注意:** 数式内のフィールド名は、実際に作成したフィールド名と完全に一致させてください。Lark Bitableの数式エディタでは、フィールド名をクリックして入力することもできます。

### 数式フィールド「人工数」の設定（詳細）

実働時間から人工数（にんくすう）を自動換算します。

**手順:**
1. 「+」ボタン → 型一覧から **「数式」** を選択

```
IF(実働時間(h) >= 7, 1.0,
  IF(実働時間(h) >= 4, 0.5, 0)
)
```

4. 「数値の形式」を **「小数」** に設定、小数点以下の桁数を **1** に設定
5. 「確定」をクリック

> **数式の解説:**
> - 実働7時間以上 → 1.0人工（1日分）
> - 実働4時間以上7時間未満 → 0.5人工（半日分）
> - 実働4時間未満 → 0人工
> - この閾値（7h / 4h）は企業ごとにカスタマイズ可能です

### 数式フィールド「金額(円)」の設定（詳細）

総人工数と作業員の日当単価を掛け合わせて金額を自動計算します。

**手順:**
1. 「+」ボタン → 型一覧から **「数式」** を選択
2. フィールド名に `金額(円)` と入力
3. 数式入力エリアに以下の数式をコピー＆ペースト

```
総人工数 * LOOKUP(作業員, 日当単価(円))
```

4. 「数値の形式」を **「整数」** に設定（小数点以下0桁）
5. 「確定」をクリック

> **数式の解説:**
> - `総人工数` : 当月の合計人工数
> - `LOOKUP(作業員, 日当単価(円))` : リンクフィールド「作業員」経由で、作業員マスタの「日当単価(円)」を参照
> - 結果は「人工数 x 日当単価」のシンプルな掛け算

> **注意:** LOOKUP関数はリンクフィールドを通じて他テーブルの値を参照します。先に「作業員」リンクフィールド（#3）を作成しておく必要があります。

> **代替方法:** LOOKUP関数がうまく動作しない場合は、以下の方法も検討してください。
> 1. 作業員マスタから「日当単価(円)」のルックアップフィールドを作成
> 2. 数式で `総人工数 * 日当単価(円)のルックアップフィールド名` とする

## Tips & トラブルシューティング

### フィールド追加時の「+」ボタンの位置

- グリッドビューの**一番右の列の右側**にあります
- 見つからない場合は、テーブルを右にスクロールしてください
- または、列ヘッダーを右クリック → 「右にフィールドを追加」でも追加可能です

### フィールド順序の変更方法

1. 変更したいフィールドの列ヘッダーをクリック＆ホールド
2. ドラッグして目的の位置に移動
3. ドロップして確定

### リンクフィールド設定時の注意点

- **リンク先テーブルが先に存在している必要があります。** 本ガイドの作成順序（T1 → T3 → T2 → T4 → T5 → T6）を守ってください
- リンクフィールドを作成すると、リンク先テーブルにも逆方向のリンクフィールドが自動作成されます。これは正常な動作であり、不要であれば「フィールドを非表示」にできます（削除はしないでください）
- 「複数レコードのリンクを許可」の設定を間違えた場合は、フィールドの設定を編集して変更可能です

### 数式の入力方法とテスト

- 数式エディタでフィールド名を入力する際は、**候補のサジェストから選択**するのが確実です
- フィールド名を直接タイプする場合は、フィールド名を正確に入力してください（全角・半角、括弧の種類に注意）
- 数式を保存した後、テストデータを1件入力して正しく計算されるか確認してください
- エラーが表示された場合は、フィールド名のスペルミスやフィールドの型が正しいか確認してください

### よくある問題と対処

| 問題 | 原因 | 対処 |
|------|------|------|
| 数式でフィールドが見つからない | フィールド名のミスマッチ | 数式エディタでサジェストからフィールドを選択 |
| リンク先テーブルが表示されない | テーブルが未作成 | 作成順序を確認し、先にリンク先テーブルを作成 |
| 自動番号が連番にならない | レコード削除後の挙動 | 自動番号は削除しても欠番を埋めません。正常動作です |
| 選択肢の色が選べない | Larkのバージョン差異 | 色は見た目の区別用です。設定できなくても機能に影響なし |
| LOOKUP関数がエラーになる | リンクフィールドが未設定 | 先にリンクフィールドを作成してから数式を設定 |
