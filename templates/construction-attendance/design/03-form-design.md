# フォーム設計書（Form Design）

建設業向け勤怠・日報テンプレート — Lark Bitable フォーム定義

---

## 設計方針

| 原則 | 詳細 |
|------|------|
| 最小操作ステップ | 出勤打刻は3ステップ（各ステップ1〜2操作）で完了 |
| 必須フィールド最小化 | 入力負荷を下げ、現場での利用率を最大化 |
| スマホファースト | 片手操作を前提とした縦長レイアウト |
| 自動取得の活用 | GPS位置情報・現在時刻はフォーム送信時に自動取得 |
| 固定値の非表示 | 打刻種別・承認状態など自明な値はフォーム上に表示しない |

---

## フォーム一覧

| # | フォーム名 | 対象テーブル | 主な利用者 | 利用頻度 |
|---|----------|------------|-----------|---------|
| F1 | 出勤打刻 | T4: 出退勤記録 | 全作業員 | 毎朝1回 |
| F2 | 退勤打刻 | T4: 出退勤記録 | 全作業員 | 毎夕1回 |
| F3 | 日報提出 | T5: 日報 | 全作業員 | 毎夕1回 |
| F4 | 承認（インライン編集） | T4 / T5 | 班長・現場監督 | 随時 |

## F1: 出勤打刻フォーム

### 基本情報

| 項目 | 値 |
|------|-----|
| フォーム名 | 出勤打刻 |
| 対象テーブル | T4: 出退勤記録（`tbl_attendance`） |
| 主な利用者 | 全作業員（スマホ） |
| 利用タイミング | 毎朝、現場到着時 |
| 目標操作数 | 3ステップ（各ステップ1〜2操作） |
| 目標所要時間 | 30秒以内 |

### フィールド構成（表示順）

| # | フィールド名 | フィールドID | 必須 | 入力方法 | デフォルト値 | フォーム上の表示 | 説明 |
|---|-------------|------------|------|---------|------------|----------------|------|
| 1 | 作業員 | `fld_att_worker` | ○ | リンク選択（T2） | -- | 表示 | 自分の名前を選択 |
| 2 | 対象案件 | `fld_att_project` | ○ | リンク選択（T1） | -- | 表示 | 本日の現場を選択 |
| 3 | 打刻種別 | `fld_punch_type` | ○ | 固定値 | 出勤 | 非表示 | フォーム送信時に「出勤」を自動セット |
| 4 | 打刻日 | `fld_att_date` | ○ | 自動入力 | TODAY() | 非表示 | 送信日の日付を自動入力 |
| 5 | 出勤時刻 | `fld_clock_in` | ○ | 時刻ピッカー | NOW() | 表示 | 現在時刻がデフォルト。タップで確定 |
| 6 | 出勤GPS | `fld_gps_in` | -- | 自動取得 | 現在地 | 非表示 | フォーム送信時にデバイスGPSを自動取得 |

### Bitable フォーム設定メモ

- 「作業員」フィールドはリンクフィールドのため、選択リストにはT2（作業員マスタ）のレコードが表示される。ステータスが「稼働中」の作業員のみ表示されるよう、ビューのフィルタ条件を設定すること。
- 「出勤GPS」はBitableの位置情報フィールドの自動取得機能を利用する。フォーム設定で「送信時に位置情報を取得」をONにする。
- 「打刻種別」「打刻日」「承認状態」はフォームに非表示とし、デフォルト値として固定値を設定する。

### Bitable フォーム設定メモ

- 作業内容の選択リストにはT3（作業内容マスタ）のレコードが表示される。「作業大分類 - 作業名」の形式で表示されるよう、T3側の主表示フィールドを設定すること。

- 承認状態は「提出済」が固定でセットされる。班長/監督がビュー上で承認ステータスを変更する運用（F4参照）。
- フォームが長いため、作業2・作業3のセクションは折りたたみ表示（条件付き表示）を推奨。作業内容2が空の場合は作業3を非表示にするとよい。

## F4: 承認フォーム（班長・現場監督用）

### 基本情報

| 項目 | 値 |
|------|-----|
| フォーム名 | -- （独立フォームではなく、ビュー上のインライン編集） |
| 対象テーブル | T4: 出退勤記録 / T5: 日報 |
| 主な利用者 | 班長（T4承認）、現場監督（T4・T5承認） |
| 利用タイミング | 毎日夕方〜翌朝 |

### 設計方針

F4は独立したフォームとして設計しない。理由は以下の通り。

1. **承認対象が複数レコード** -- 班長/監督は一度に複数名分の出退勤・日報を確認するため、一覧ビューから直接操作する方が効率的。
2. **文脈の確認が必要** -- 打刻時刻や作業内容を確認した上で承認するため、レコードの全フィールドを閲覧できるビュー上での操作が適切。
3. **Bitableの機能活用** -- Lark Bitableはビュー上でのインライン編集に対応しており、単一選択フィールドのステータス変更をワンタップで行える。

### 承認対象フィールド

#### T4: 出退勤記録の承認

| フィールド名 | フィールドID | 操作 |
|-------------|------------|------|
| 承認状態 | `fld_att_approval` | 単一選択を変更: 未承認 → 班長承認 → 監督承認 |
| 備考 | `fld_att_note` | 差戻しの場合、理由を追記 |

#### T5: 日報の承認

| フィールド名 | フィールドID | 操作 |
|-------------|------------|------|
| 承認状態 | `fld_report_approval` | 単一選択を変更: 提出済 → 班長承認 → 監督承認 |
| 承認コメント | `fld_approval_comment` | 差戻し理由やフィードバックを記入 |

### Bitable 設定メモ

- 班長ユーザーには該当テーブルのフィールド編集権限を付与する。ただし「承認状態」「承認コメント」「備考」以外のフィールドは編集不可とし、打刻データの改ざんを防止する。
- 監督ユーザーには全フィールドの閲覧権限と「承認状態」「承認コメント」の編集権限を付与する。
- 差戻し時は自動化ルールにより、対象の作業員にLark通知を送信する設計（自動化設計書で詳細を定義）。

## 共通設計事項

### フォームURLの共有方法

| 方法 | 手順 | 推奨度 |
|------|------|--------|
| QRコード | フォーム公開URLからQRコードを生成し、現場事務所に掲示 | 高 |
| Larkブックマーク | 各作業員のLarkアプリにフォームURLをブックマーク登録 | 最高 |
| Larkグループチャット | 現場グループチャットにフォームリンクをピン留め | 高 |
| ホーム画面ショートカット | スマホのホーム画面にWebショートカットとして追加 | 高 |

### ブックマーク登録手順（推奨）

```
1. Larkアプリを開く
2. 対象フォームのURLを共有リンクとして取得
3. Larkアプリ内のブックマーク（お気に入り）に登録
4. 以下の3つを登録：
   - 出勤打刻（F1）
   - 退勤打刻（F2）
   - 日報提出（F3）
```

### フォーム送信後の動作

| 項目 | 設定内容 |
|------|---------|
| 送信完了メッセージ | 「送信が完了しました」を表示 |
| 送信後の遷移 | 同じフォームを再表示（翌日の利用に備える） |
| 確認通知 | 自動化ルールにより、送信者にLark通知を送信（任意設定） |

### 二重送信防止

Lark Bitableフォームには標準の二重送信防止機能がないため、以下の運用ルールで対処する。

| 対策 | 詳細 |
|------|------|
| 運用ルール | 「送信完了メッセージが表示されるまでボタンを連打しない」旨を周知 |
| 自動化チェック | 自動化ルールにより、同日・同一作業員・同一案件の重複レコードを検出し、管理者に通知 |
| 管理者対応 | 重複レコードが発生した場合、管理者がビュー上で手動削除 |

### フォーム公開設定

| 設定項目 | F1: 出勤打刻 | F2: 退勤打刻 | F3: 日報提出 |
|---------|-------------|-------------|-------------|
| 公開範囲 | 組織内メンバーのみ | 組織内メンバーのみ | 組織内メンバーのみ |
| 送信回数制限 | 制限なし | 制限なし | 制限なし |
| 送信後フォーム再表示 | ON | ON | ON |
| 送信完了メッセージ | 「出勤打刻が完了しました」 | 「退勤打刻が完了しました」 | 「日報を提出しました」 |

### 入力補助のヒント

作業員がフォームを使いやすくするためのフィールド説明文（ヘルプテキスト）を設定する。

| フォーム | フィールド | ヘルプテキスト |
|---------|----------|--------------|
| F1 | 作業員 | 「自分の名前を選択してください」 |
| F1 | 対象案件 | 「本日の現場を選択してください」 |
| F1 | 出勤時刻 | 「現在時刻が入っています。修正がなければそのまま送信してください」 |
| F1 | 天候 | 「任意です。入力しなくても送信できます」 |
| F2 | 退勤時刻 | 「現在時刻が入っています。修正がなければそのまま送信してください」 |
| F2 | 休憩時間 | 「昼休み1時間の場合はそのまま。変更がある場合のみ修正してください」 |
| F3 | 作業内容1 | 「本日のメインの作業を選択してください」 |
| F3 | 作業時間1 | 「時間単位で入力（例: 7.0、3.5）」 |
| F3 | 写真 | 「現場の写真があればカメラで撮影して添付してください（任意）」 |
