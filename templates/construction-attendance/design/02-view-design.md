# ビュー設計書（View Design）

建設業向け勤怠・日報テンプレート — Lark Bitable ビュー定義

---

## ビュー設計方針

### 1. 役割別最適化

本テンプレートでは3つの役割を想定し、それぞれに最適化されたビューを設計する。

| 役割 | 主な利用シーン | 端末 | 求められる体験 |
|------|--------------|------|--------------|
| **職人** | 出退勤打刻、日報入力、自分の勤怠確認 | スマホ中心 | 最小タップで完結、今日のデータだけ見える |
| **班長** | 班員の出退勤確認、承認、日報チェック | スマホ + PC | 未承認一覧に素早くアクセス、班単位で把握 |
| **管理者（現場監督・事務）** | 全体俯瞰、集計確認、マスタ管理 | PC中心 | 全データ閲覧、フィルタ・ソートで分析 |

### 2. スマホ閲覧への配慮

- 表示フィールドは**最大5〜7列**を目安とし、横スクロールを最小限に抑える
- 最も重要な列（氏名・日付・ステータス等）を左側に配置する
- カレンダービュー・カンバンビューはスマホでも視認性が高いため積極的に活用する
- グリッドビューでは「行の高さ: 小」を基本とし、一覧性を確保する

### 3. デフォルトビューの考え方

- 各テーブルの**最初のビュー（デフォルト）は最も高頻度で使用されるもの**を設定する
- トランザクションテーブル（T4, T5）は「本日のデータ」をデフォルトとする
- マスタテーブル（T1, T2, T3）は「全件一覧」をデフォルトとする

## T1: 案件マスタ（`tbl_projects`）

### V1-1: 全案件一覧

| 項目 | 内容 |
|------|------|
| **ビュー名** | 全案件一覧 |
| **ビュー種類** | グリッド |
| **デフォルト** | Yes（テーブルを開いたとき最初に表示） |
| **表示フィールド** | 1. 案件ID / 2. 案件名 / 3. ステータス / 4. 現場住所 / 5. 発注者 / 6. 工期開始 / 7. 工期終了 / 8. 現場監督 / 9. 備考 |
| **非表示フィールド** | 現場GPS緯度 / 現場GPS経度 / GPS許容半径(m) |
| **フィルタ条件** | なし（全件表示） |
| **ソート条件** | ステータス 昇順（準備中 > 進行中 > 完了 > 中止）、案件名 昇順 |
| **グループ化** | なし |
| **主な利用者** | 管理者 |
| **用途** | 全案件の俯瞰管理。マスタデータの閲覧・編集。ステータスのフィルタ操作で特定状態の案件を絞り込み可能。 |

### V1-2: 進行中案件

| 項目 | 内容 |
|------|------|

## ビュー権限設計（参考）

Lark Bitableの共有権限設定と組み合わせて、役割別のアクセスを制御する。

| 役割 | 閲覧可能テーブル | 編集可能テーブル | 備考 |
|------|----------------|----------------|------|
| **職人** | T4（自分のみ）, T5（自分のみ） | T4（自分のレコード追加のみ）, T5（自分のレコード追加のみ） | フォーム経由での入力を基本とする |
| **班長** | T1, T2（単価除く）, T4, T5 | T4（承認状態の変更）, T5（承認状態の変更） | 班員のデータに対する承認権限を持つ |
| **管理者** | 全テーブル | 全テーブル | フル権限。マスタ管理・集計確認・最終承認を担当 |

> **注意:** Lark Bitableのフィールド単位の権限制御には制約があるため、日当単価等の機密フィールドはビューの非表示設定で対応する。厳密な権限制御が必要な場合は、テーブルを分離するか、Larkの権限グループ設定を併用すること。

## スマホ向けビュー推奨設定

スマートフォンからの閲覧・操作を快適にするための推奨設定をまとめる。

| 設定項目 | 推奨値 | 理由 |
|---------|--------|------|
| 行の高さ | 小（Low） | 一覧性を確保し、スクロール量を削減 |
| 表示列数 | 5〜7列以内 | 横スクロールなしで主要情報を把握 |
| 固定列 | 先頭1〜2列を固定 | 氏名・日付を常に表示 |
| フィールド幅 | テキスト: 120px / 日付: 100px / 数値: 80px | 最小限の幅で情報を収める |
| デフォルトビュー | フィルタ済みビュー | 「今日」「未承認」など、すぐに使えるビューをデフォルトに |
